*1. Setting a Filter (With Priority)*
Create a trigger and a response. We *highly recommend* setting a Priority (`?pr`) for every filter.

*Why use Priority?*
If a message matches two different filters, the bot will only send the one with the *highest* priority number.
> â€¢ If you do not set `?pr`, it defaults to *0*.
> â€¢ If a Priority 0 filter and a Priority 10 filter both match a message, the Priority 0 filter will *never* run.

`.filter <trigger>`
`?pr <number>`
`! <response>`

> â€¢ *Example:*
> `.filter price`
> `?pr 10`
> `! The price is $10.`
> â€¢ _Explanation:_ Sets the filter with Priority 10. This ensures it overrides any default (0) filters that might also match.

*Multiple Responses (Random)*
You can add multiple `!` lines. The bot will pick one randomly each time.

> â€¢ *Example:*
> `.filter hi`
> `?pr 5`
> `! Hello there!`
> `! Hi friend!`
> `! Greetings.`

*2. Deleting a Filter*
Remove a specific trigger so the bot stops replying to it.

`.delflt <trigger>`

> â€¢ *Example:* `.delflt price`
> â€¢ _Explanation:_ Deletes the filter for "price".
> â€¢ *Example:* `.delflt "phrase with spaces", otherword`
> â€¢ _Explanation:_ Deletes two filters at once. Use quotes if the trigger has spaces.

*4. Typing Simulation (Realism)*

`.filterdelay <seconds>`

> â€¢ *Example:* `.filterdelay 2`
> â€¢ _Explanation:_ When a filter is triggered, the bot waits 2 seconds (showing "Typing...") before sending the message.

`.filter <command> jd:groupName`

> â€¢ *Example:* `.filter rules jd:Support`
> `?pr 99`
> `! Read the pinned message.`
> â€¢ _Explanation:_ Sets a high-priority "rules" filter for the "Support" group.
> â€¢ *Example:* `.delflt spam jd:all`
> â€¢ _Explanation:_ Deletes the "spam" filter from *all* groups.
